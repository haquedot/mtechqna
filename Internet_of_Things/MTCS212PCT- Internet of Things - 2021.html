IoT Concept, History & Relation with WSN

5/10 marks
// definition

Internet of Things (IoT) is a network of physical objects ("things") embedded with sensors, software, and connectivity, enabling them to collect and exchange data over the internet вҖ” without human-to-human or human-to-computer interaction.

Origin & History

  • Coined by: Kevin Ashton in 1999 at MIT Auto-ID Center
  • First practical IoT device: Internet-connected Coca-Cola vending machine (1982, Carnegie Mellon University)
  • 2000s: RFID-based tracking systems became mainstream
  • 2010s: Explosion of smart devices вҖ” smartphones, wearables, smart home
  • Today: 15+ billion connected devices globally; expected to exceed 30 billion by 2030

Wireless Sensor Network (WSN) вҖ” Definition

// WSN definition

A Wireless Sensor Network (WSN) is a network of spatially distributed autonomous sensor nodes that monitor physical or environmental conditions (temperature, pressure, motion) and wirelessly transmit data to a central point (sink/base station).

Difference between IoT and WSN

FeatureIoTWSN
ScopeBroad вҖ” any connected device/systemNarrow вҖ” specifically sensor-based networks
Internet AccessMandatory вҖ” devices connect to internetOptional вҖ” may be local only
IntelligenceHigh вҖ” AI, cloud computing, automationLow to medium вҖ” data collection focused
PowerOften mains-powered or rechargeableBattery-powered, ultra-low power
Data ProcessingEdge + Cloud (distributed)Sink node / base station (centralized)
ExamplesSmart home, connected car, wearablesForest fire detection, battlefield monitoring
ScaleBillions of devices globallyLimited area deployments

How WSN and IoT Relate

WSN is a subset/component of IoT. Every WSN can be part of an IoT system, but IoT is much broader than WSN.

IoT is an ecosystem where physical devices connect to the internet to collect, share, and act on data intelligently. WSN forms the foundational sensing layer of IoT вҖ” providing raw environmental data that IoT systems process in the cloud to deliver smart applications.

How Arduino Works (Step-by-Step)

1
Write Code in Arduino IDE

Write a sketch (program) in C/C++ syntax. Functions: setup() runs once; loop() runs forever.

2
Compile the Sketch

IDE compiles code into machine-level hex file for the ATmega328P microcontroller.

3
Upload via USB

Compiled code is uploaded via USB using the onboard USB-to-Serial chip (ATmega16U2).

4
Sensor Data Collection

Sensors connected to analog/digital pins send data; Arduino reads values using analogRead()/digitalRead().

5
Processing & Decision

ATmega328P processor executes code вҖ” makes decisions (if temperature > 30, turn fan ON).

6
Actuator Control / Communication

Output commands sent to actuators (motors, LEDs) or data sent via Serial/WiFi module to cloud.

// conclusion

Arduino is the most beginner-friendly IoT development platform. Its simple programming model (setup + loop), abundant libraries, and extensive community support make it ideal for rapid prototyping of IoT solutions from smart home devices to industrial sensors.

Sensors, Actuators & Classification of Sensors

5/10 marks
// sensor definition

A Sensor is a device that detects changes in the physical environment (temperature, pressure, light, motion) and converts them into electrical signals that can be read by a microcontroller or computer.

// actuator definition

An Actuator is a device that receives electrical/digital signals from a controller and converts them into physical actions вҖ” movement, heat, sound, or light. Actuators are the "output" side of IoT.

Q06

IoT-Based Cloud Computing

5/10 marks
// definition

IoT-based Cloud Computing is the integration of IoT devices with cloud infrastructure for storing, processing, and analyzing the massive amounts of data generated by IoT devices. Cloud provides scalable, on-demand resources that IoT devices cannot handle locally.

Why Cloud is Essential for IoT

  • IoT devices are resource-constrained (limited CPU, RAM, storage)
  • Data volume is enormous вҖ” billions of sensor readings per day
  • Cloud provides unlimited scalable storage and compute power
  • Cloud enables real-time analytics, ML model execution, and alerting

2. CoAP (Constrained Application Protocol)

  • Lightweight REST-like protocol for constrained IoT devices
  • Works over UDP (unlike HTTP which uses TCP)
  • Supports GET, POST, PUT, DELETE (like HTTP)
  • Use case: Smart energy meters, embedded sensors with very limited resources

3. HTTP / HTTPS

  • Standard web protocol вҖ” used when IoT devices have enough resources
  • RESTful APIs are commonly used for IoT cloud platforms
  • Limitation: Too heavy for battery-powered constrained devices

4. AMQP (Advanced Message Queuing Protocol)

  • Enterprise-grade messaging protocol with reliability features
  • Supports message acknowledgment, transactions, security
  • Use case: Industrial IoT, financial IoT applications

5. WebSocket

  • Full-duplex (two-way simultaneous) communication over a single TCP connection
  • Used for real-time IoT dashboards and live monitoring

Interoperability in IoT

// interoperability definition

Interoperability is the ability of different IoT devices, platforms, and systems from different manufacturers to communicate, exchange data, and work together seamlessly without special adaptation.

Why Interoperability is a Problem in IoT:

  • IoT ecosystem has hundreds of vendors with proprietary protocols
  • A Samsung smart TV cannot natively talk to an LG smart AC
  • Different physical layers (WiFi, Zigbee, Z-Wave) don't communicate directly
  • Data formats vary (JSON, XML, proprietary binary) across platforms

Levels of Interoperability:

LevelNameDescriptionSolution
1TechnicalPhysical + data link connectivityStandardized radio protocols (Zigbee, Matter)
2SyntacticData format compatibilityJSON, XML standards; REST APIs
3SemanticShared meaning/understanding of dataOntologies, W3C WoT standards
4OrganizationalCross-organization data sharingData governance policies, open APIs

Solutions for Interoperability:

  • Matter Protocol: New unified smart home standard by Apple, Google, Amazon, Samsung
  • oneM2M: Global IoT interoperability standard for M2M communications
  • W3C Web of Things (WoT): W3C standard for semantic IoT descriptions
  • IoT Gateways: Act as protocol translators between incompatible devices
  • Middleware platforms: OpenIoT, FIWARE вҖ” enable cross-platform integration

Challenges in IoT

SecurityBillions of devices are attack targets; weak passwords, unpatched firmwareEnd-to-end encryption, TLS, regular OTA updates

IoT Security Threats (Exam Focus):

  • Default Credentials: Many IoT devices ship with "admin/admin" passwords
  • Man-in-the-Middle: Attacker intercepts unencrypted IoT data
  • Firmware Attacks: Malicious firmware updates pushed to IoT devices
